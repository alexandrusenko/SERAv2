- Легковесный fallback: **qwen2.5-14b-instruct-q5_k_m.gguf**.

## Паттерны реализации

### 1) Pure Tool Calling
Плюсы: контроль, безопасность, предсказуемость.
Минусы: потолок возможностей = список tools.

### 2) Pure Code-per-task
Плюсы: гибкость, «безлимит» функций.
Минусы: высокий риск нестабильности, сложнее безопасность и дебаг.

### 3) Hybrid (рекомендуется)
База: Tool Calling.
Расширение: on-demand генерация нового tool (plugin), проверка AST, hot-load.

Это лучший компромисс между управляемостью и адаптивностью.
